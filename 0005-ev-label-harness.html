1. 今天的核心模型

EV label = 对一个决策分支的长期收益标签。

如果 branch eval 是“跑两条路”，那么 EV label harness 就是把很多分支结果整理成可训练、可筛选、可复查的标签库。

branch rollout result
  -> EV label row
  -> label quality report
  -> shadow / candidate / gate

2. 英文术语速查

英文术语中文理解在本项目里的意思
EV期望收益长期平均得分或收益，不是单局运气。
label标签给某个决策分支打上的收益结果，例如 peng 比 pass 高多少。
score_delta分数差候选分支平均分 - 当前分支平均分。
positive_rate正样本比例多少比例的 label 支持候选分支。
median中位数把结果排序后中间的值，比平均数更不容易被极端值骗。
outlier极端值少数特别大的结果，可能把平均数拉高。
fresh label新鲜标签来自新语料/新 run 的标签，比历史重复样本更能检验泛化。
max_turns最大模拟回合数正式 claim/pass EV 证据必须是 180，不能用短截尾当结论。
promotion_evidence晋级证据是否足够支持进入 profile gate；本项目默认先是 false。

4. 为什么不能只看平均数

假设 5 个 label 的 score_delta 是：

+20, -2, -2, -2, -2

平均数是 +2.4，看起来正；但中位数是 -2，大部分样本其实反对候选。这就是 outlier-driven positive：被一个极端正例拉高。

判断 EV label 稳定性，至少看三件事：

  • 平均数是否正；
  • 中位数是否正；
  • positive_rate 是否足够高。

如果平均数正但中位数 ≤ 0，就要高度怀疑是 outlier 驱动。

5. 读 label_quality_report 的顺序

  1. 先看 max_turns：claim/pass 正式证据是否全是 180？
  2. 再看 label_count：是不是太少？
  3. 再看 median_score_delta：中位数是否支持候选？
  4. 再看 positive_rate：多数 label 是否同方向？
  5. 最后看 outlier：是不是被一个极端值拉起来？
