1. 今天的核心模型

A/B 是整体安全检查；Profile Gate 是写入审批门。

在前几课里，你已经学了 branch eval、changed-choice、EV label、shadow。A/B 与 profile gate 通常应该排在后面：它们不是发现真理的第一步，而是检查“这个候选改到整体游戏里是否安全、是否值得人工批准写入”。

candidate contract
  -> changed-choice audit
  -> changed-row EV
  -> shadow / boundary
  -> whole-game A/B safety
  -> manual profile gate

英文术语中文理解在本项目里的意思

candidate候选版本准备验证的策略改动、profile patch 或模型输出。
whole-game整局游戏不是只看目标分支，而是让 AI 正常打完整局。

profile gate策略配置写入门决定是否允许请求写 ai_profile_train/play.json。

play profile实战配置data/ai_profile_play.json，影响移动端/实战建议，更严格。

4. 为什么 A/B 不是万能的

whole-game A/B 很重要，但它不能替代前面的证据。

  • changed-choice 太少：A/B 可能几乎不触发候选，结果接近 0。
  • changed-choice 太宽：A/B 负了也不知道伤害来自哪类局面。
  • EV label 不稳：A/B 可能只是把噪声放大。
  • 没有 rollback：即使 A/B 小正，也不应该轻易写 play profile。

所以正确心智是：

A/B 不是“证明单点策略正确”的工具；
A/B 是“候选进入整体系统后是否安全”的工具。
