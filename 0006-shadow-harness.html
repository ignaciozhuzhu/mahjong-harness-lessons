1. 今天的核心模型

Shadow = 只在旁边打分，不接管真实动作。

Shadow harness 用来回答：“这个模型/规则的判断，和 EV label 的真实收益方向是否一致？”如果一致，才可能进入 candidate；如果不一致，只能作为 diagnostic。

EV label bank
  -> shadow scoring / calibration
  -> candidate_ready 或 diagnostic_only
  -> 仍然不能直接改动作

2. 英文术语速查

英文术语中文理解在本项目里的意思
shadow影子模式模型/规则只旁路评分，不改变真实 AI 行为。
offline scoring离线评分在已有数据/标签上打分，不进入实战决策。
calibration校准检查模型信心与真实 EV 是否方向一致。
confidence信心分shadow 认为某个选择更好的强度。
correlation相关性confidence 越高，真实 EV 是否也越好。
candidate_ready候选准备好可以进入 candidate contract / changed-choice，而不是直接上线。
diagnostic_only仅诊断有观察价值，但不能用于改动作或写 profile。
action mutation动作改写模型直接改变 AI 打哪张/吃碰过；shadow 阶段必须禁止。
quality gate质量关卡根据 label 数、fresh label、median、positive_rate、correlation 判断是否过关。

4. 为什么 shadow 不能直接改动作

因为 shadow 的目标是“判断是否值得继续”，不是“已经证明可以上线”。

一个 shadow 模型即使看起来正向，也可能只是：

  • 在历史样本上过拟合；
  • 只会找少数极端正例；
  • confidence 和真实 EV 没有正相关；
  • 会在自然局面里改太多动作。

所以 shadow 阶段必须保持：action_mutation_allowed=false。

5. 一个真实结果怎么读

示例报告：/Users/maccio/work/hangzhou-mahjong-trainer/data/shadow_diagnostic/20260617_claim_peng_refined_split_ev/shadow_calibration_report.json

claim:peng
label_count: 174
fresh_avg_score_delta: +1.2803
fresh_median_delta: +0.4
fresh_positive_rate: 0.5632
recommendation: diagnostic_only
candidate_review_ready_count: 0

这说明它不是完全没信号：平均数和中位数略正。但 positive_rate 只有约 56%，仍然有大量负例/噪声，所以结论是 diagnostic_only，不是 candidate ready。
