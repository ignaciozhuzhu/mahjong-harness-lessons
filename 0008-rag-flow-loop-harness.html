1. 今天的核心模型

RAG Flow Loop = 管理所有 harness 证据的元 harness。

前面几课讲的是单个 harness：branch eval、changed-choice、EV label、shadow、A/B、profile gate。第 8 课讲的是：这些证据如何进入一个统一流程，让 agent 不再凭感觉调参。

opportunity discovery
  -> 十问 Improvement Direction Gate
  -> evidence registry / failure ledger
  -> RAG summary pack
  -> next gate decision

2. 英文术语速查

英文术语中文理解在本项目里的意思
RAG检索增强生成让 agent 先检索项目记录，再回答/决策，避免忘记历史失败。
flow loop流程闭环发现候选、评估证据、分流下一步、写入记录，再继续。
opportunity机会 / 候选方向可能能提升 AI 的调参方向或错误家族。
ten-question gate十问关卡用 10 个问题判断候选该 reject、diagnostic、label、shadow 还是 candidate。
evidence registry证据登记表把 gate、score、failure ledger 等证据统一索引。

summary pack摘要包压缩当前训练状态，供 RAGFlow 检索。
route分流路线reject、diagnostic_only、needs_label、needs_shadow、candidate_review_ready。
read-only只读只生成报告，不写 profile，不部署。

4. 十问 gate 如何串起前 7 课

十问字段来自哪一课作用
evidence_scope第 1-2 课判断是单点正例还是错误家族。
coverage_count第 4 课判断覆盖够不够。
runtime_visible第 2-3 课确认策略只使用当时可见信息。
negative_boundary_status第 4-6 课确认是否知道哪些局面会伤害。
changed_choice_count第 4 课判断候选到底会改多少真实决策。
ev_label_stability第 5 课判断 EV label 是否稳定。

whole_game_ab_status第 7 课判断整体安全性。
profile_write_recommendation第 7 课判断是否只能人工 gate。
final_route第 8 课决定下一步进入哪条路线。

5. 为什么这是“元 harness”

普通 harness 回答一个局部问题，比如“peng 比 pass 好吗？”

RAG Flow Loop 回答的是更高层问题：

这个方向是否值得继续花预算？
是否重复了历史失败？
缺的是 changed-choice、EV label、shadow 还是 A/B？
是否只能 diagnostic，还是可以 candidate review？

所以它不是替代 branch eval / EV label / A/B，而是管理这些证据的流程外壳。
