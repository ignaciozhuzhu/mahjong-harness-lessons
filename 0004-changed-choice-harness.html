第 4 课：Changed-choice Harness

目标：理解为什么“策略看起来正向”还不够，必须先知道它到底会改变多少真实决策。

Harness 课程导航： 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8
完整课程总索引

1. 今天的核心模型

Changed-choice = 候选策略真的会把当前策略的哪些决策改掉。

很多候选在目标 case 上看起来很强，但放到真实 replay / self-play 决策语料里，可能几乎不触发；也可能触发太广，改了一堆不该改的局面。Changed-choice harness 就是先检查这个。

2. 为什么 changed-choice 很关键

情况风险应该怎么处理
changed rows = 0候选几乎不改变行为，whole-game A/B 很可能看不到收益。停止 profile 路线，保留 diagnostic 或重新设计触发条件。
changed rows 很少可能只是 exact patch，目标 case 翻正但长期强度无意义。需要证明这些 changed rows 的 EV 很高；A/B 只当安全检查。
trigger rate 太高候选太宽，可能误伤大量自然局面。先收窄边界，找负例排除条件。
changed rows 适中且可解释最适合进入 changed-row EV。生成 EV label queue，检查正负边界。

2.5 英文术语速查

本课会出现一些英文项目术语，先看这张表：

英文术语中文理解在本项目里的意思
changed-choice改变了选择候选策略相比当前策略，实际会把哪些决策改掉。
changed rows被改变的行数在决策语料里，有多少条决策会被候选策略改写。
trigger rate触发率changed rows / audited opportunities；表示候选策略触发得多不多。
audited opportunities被审计的机会数总共扫描了多少个可能触发候选策略的局面。
candidate候选策略还没有上线，只是在验证中的规则、profile 参数或 contract。
contract候选约束说明明确候选策略允许改哪些局面、不能改哪些局面。
exclusion counts排除数量统计说明有多少局面因为 BB、七对倾向、chi overlap 等原因被排除。
gate关卡 / 审批门决定候选下一步是 reject、diagnostic、EV label、shadow、A/B 还是 profile gate。
diagnostic_only仅诊断有学习价值，但证据不够，不能写 profile。
profileAI 策略配置data/ai_profile_train.json / data/ai_profile_play.json 这类控制 AI 行为的配置。

3. 本项目里的真实例子

看 /Users/maccio/work/hangzhou-mahjong-trainer/data/candidate_reviews/20260617_claim_peng_refined_split_audit/changed_choice_audit_report.md：

Audited opportunities: 9284
Changed rows: 94
Trigger rate: 0.010125
Next gate decision: ready_for_changed_row_ev

这说明 refined claim:peng contract 不是只改 1 个目标 case，也不是大范围乱改。它在 9284 个机会里改了 94 个，触发率约 1.01%，因此适合进入下一步 changed-row EV，而不是直接 A/B 或写 profile。

4. 四件套拆解

Harness 四件套Changed-choice audit 里的对应物
Input候选 contract / candidate profile + replay/self-play decision corpus
Procedure逐行比较 current policy 与 candidate policy：当前 pass 是否会变 peng？当前 discard 是否会变另一张？
Metricsaudited opportunities、changed rows、trigger_rate、exclusion counts、changed_by_family
Decision boundary决定是 reject / diagnostic_only / ready_for_changed_row_ev / too_broad_requires_refinement

5. 关键误区

不要把 whole-game A/B 当成 changed-choice 的替代品。

如果 changed-choice 太低，A/B 可能跑了很多局也几乎没改动作，结果接近 0 并不能证明策略没用；它可能只是低触发。如果 changed-choice 太高，A/B 负了也很难知道是哪类 changed rows 造成伤害。

正确顺序通常是：

candidate contract
  -> changed-choice audit
  -> changed-row EV
  -> negative boundary split
  -> whole-game safety A/B
  -> manual profile gate

6. 小测验：怎么判断 changed-choice 结果？

一个候选在目标 case 上 EV 很正，但 changed-choice audit 显示：扫描 9000 个机会，只改变 1 行。最合理的结论是？
